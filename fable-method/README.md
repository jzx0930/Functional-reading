# fable-method

- 連結：https://github.com/Sahir619/fable-method
- 作者：Sahir619
- 分類：AI Agent Skills / 工作流方法論
- 一句話用途：把 Claude Fable 5 那套「一氣呵成」的工作流逆向拆解、蒸餾成任何模型都能安裝執行的 skill，並附上一套 eval 來確保品質不走鐘。
- 來源貼文：Threads [@leeunhn](https://www.threads.com/@leeunhn/post/Da4a-MhiQXu)（5,104 瀏覽）

## 核心概念

大家以為 Fable 5 那種一氣呵成是「模型多神」，但作者把整套工作流逆向拆開，做成一包誰都能裝的 skill——宣稱連能力弱一點的模型照著跑，也能複製那個手感。兩週衝到 1,314 顆星。

方法論可濃縮成三步：**Think / Act / Prove（想 / 做 / 證明）**。repo 說明強調「distilled into skills any model can run, with the eval that keeps it honest」——附帶的 eval 是用來持續驗證流程有沒有真的做到位。

## 重點啟示

- 很多時候「不是模型贏，是流程贏」：把高手的動作拆成步驟寫死，換個引擎照樣跑，往往比換更貴的模型更划算。

## 使用建議 / 注意事項（含留言區觀點）

- 留言區有反面聲音值得一併記著：
  - 有人認為「流程贏」有其極限——門檻低的東西別人也能靠 try-and-error 最佳化，真正的差異可能還是在模型來源／能力。
  - 也有人指出這類「綁定特定模型的 prompt/skill」有時效性，模型更新後可能被視為理所當然或被忽略。
- 綜合：把它當成「可攜、可驗證的工作流範本」來用，而不是把某個模型的優勢神化。
