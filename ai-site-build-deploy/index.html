AI 自架網站 + 部署 + SEO 流程

工作流方法

用訂閱制 AI（Codex／Claude Code）自己做出產品網站，串接 GitHub + Vercel + Cloudflare 自動部署，再用 SEO 帶自然流量。

整體流程

Codex / Claude Code 做站→ GitHub 存碼→ Vercel 自動部署→ Cloudflare 網域 + DNS
  1. 請 Codex／Claude Code 幫你做網站，跟它討論想要的樣子。
  2. 申請 GitHub，讓 AI 幫你把程式碼同步上去。
  3. 申請 Vercel，綁定 GitHub 專案，一有改動就自動重新部署。
  4. 買網域（作者用 Cloudflare，便宜好用）。
  5. 在 Vercel 加自訂網域、設定 DNS（較複雜，可請 AI 協助）。

懶人合併版：直接跟 AI 說「我有 XXX 產品，想做產品行銷網站，不需後台，需串接 GitHub、Vercel 自動部署」，剩下反覆討論到順眼。

觀念：三層架構（別把 GitHub Pages 跟 Cloudflare 搞混）

  • GitHub（repo）：存原始碼
  • 網站主機（GitHub Pages 或 Vercel）：把程式碼變成能打開的網站
  • Cloudflare：買網域＋管 DNS

真正「二選一」的是 GitHub Pages vs Vercel（誰當主機）；Cloudflare 是網域層，兩種主機都能掛。

GitHub PagesVercel
適合純靜態靜態 + 動態(SSR/serverless)
自動部署可（Actions）原生，含 preview
商業用途可免費 Hobby 僅限非商業
費用免費有免費額度

SEO：帶自然流量

  1. 設定與尋找關鍵字（含長尾）。
  2. 用關鍵字請 AI 寫對應文章。
  3. 文章一定要「去 AI 化」：作者推薦工具 better-writing。
  4. 把 sitemap 交給 Google Search Console，依錯誤調整。

注意事項（留言）

最容易卡「網域」那步：Cloudflare 改了 DNS，但 Vercel 沒抓到正確 CNAME，會一直 pending；要回 Vercel 重新驗證一次。此坑無論主機選誰、只要用自訂網域都可能遇到。
