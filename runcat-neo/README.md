# RunCat Neo

- 連結：https://github.com/runcat-dev/RunCatNeo
- 分類：macOS 工具 / 選單列 / 用量監控
- 一句話用途：macOS 選單列上會跑步的貓咪，新版可透過 Custom Metrics 直接顯示 Claude Code / Codex 的用量，不用切視窗或開網頁。
- 來源貼文：Threads [@iluciddreaming](https://www.threads.com/@iluciddreaming/post/Da8Kmo6myCf)（4,664 瀏覽）

## 它是什麼

RunCat 是知名的 macOS 選單列小工具（一隻會隨 CPU 忙碌程度加速奔跑的貓）。新版 **RunCat Neo** 開源，並新增「**Custom Metrics**」外掛，可以在選單列貓咪旁掛一張卡片，即時顯示自訂指標。

## 對 Claude Code / Codex 使用者的重點

透過 Custom Metrics（可搭配 Claude Code 的 statusline 功能把 session limit 整合進來），選單列就能即時看到：

- 目前用的 model
- context window 用了多少
- 5 小時／7 天額度還剩多少

瞄一眼選單列就知道，不用切視窗、不用開網頁。

留言補充：context window 用量越高，代表 Claude 能記住的對話歷史越少；太滿（約 83%～84%）時可能需要壓縮舊內容或開新對話。

## 適用對象

用 Claude Code／Codex 的 macOS 使用者（順便是愛貓人士——選單列住一隻跑步的貓很療癒）。
